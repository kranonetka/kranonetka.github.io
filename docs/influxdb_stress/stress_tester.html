read(nodes_count: int, aggregation: str = 'mean', type: str = 'float', start_date: Union[datetime.datetime, str] = 'now() - 5m', end_date: Union[datetime.datetime, str] = 'now()', time_interval: str = '5s') → Tuple[float, dict][исходный код]¶

Одновременное чтение несколькими потоками

Параметры

  • nodes_count – Количество одновременно читающих узлов (потоков)

  • aggregation – Функция агрегации, применяемая к группам. По умолчанию mean

  • type – Тип читаемых значений. По умолчанию float

  • start_date – Начало временного окна. Задаётся в виде datetime объекта, либо строкой по правилам синтаксиса InfluxDB (см. https://docs.influxdata.com/influxdb/v1.8/query_language/explore-data/#time-syntax)

  • end_date – Конец временного окна. Задаётся в виде datetime объекта, либо строкой по правилам синтаксиса InfluxDB (см. https://docs.influxdata.com/influxdb/v1.8/query_language/explore-data/#time-syntax)

  • time_interval – Временной интервал для группировки. Задаётся строкой по правилам синтаксиса Influxdb (см. https://docs.influxdata.com/influxdb/v1.8/query_language/spec/#durations)

Результат

Время (в секундах), прошедшее с момента одновременного начала чтения данных каждым потоком до момента получения ответа каждым из потоков и результат выборки

Примечание: выбираются записи с любыми тегами

write(nodes_count: int, float_sensors: int = 1, int_sensors: int = 1, str_sensors: int = 1, bool_sensors: int = 1, duration: int = 1, start_date: datetime.datetime = None) → float[исходный код]¶

Одновременная запись несколькими потоками

Параметры

  • nodes_count – Количество одновременно пишущих узлов (потоков)

  • float_sensors – Количество вещественных датчиков на узле

  • int_sensors – Количество целочисленных датчиков на узле

  • str_sensors – Количество строковых датчиков на узле

  • bool_sensors – Количество булевых датчиков на узле

  • duration – На протяжении скольки секунд копились данные для записи каждым узлом

  • start_date – Начиная с какой даты вести запись. По умолчанию - локальная дата запуска метода

Результат

Время (в секундах), прошедшее с момента одновременного начала отправки данных каждым потоком до момента получения ответа каждым из потоков

Примечание: Каждый поток подготавливает и держит свои данные для отправки в памяти. При генерации больших объемов будет много времени затрачено на саму генерацию и много памяти будет отведено под хранение, пока остальные потоки не подготовят свои данные
